## HTTP Traffic Tap (claude-tap-style)

### ✅ Per-session HTTP tap for remote agent CLIs
- remote-agent-host src/tap/: MITM (HTTPS_PROXY + rcgen CA, decrypt all HTTPS) AND
  reverse (ANTHROPIC_BASE_URL/base-URL injection) modes; one proxy per session on 127.0.0.1
- HttpExchange protocol msg → demux relay emits http:traffic + appends JSONL to
  ~/.remote-ai-ide/traces/; "HTTP 流量" bottom-panel tab with request/response detail
- TLS backend = ring (not aws-lc) to keep cargo xwin Windows cross-build working
- Auth headers (authorization/x-api-key/cookie) redacted at the source before send
- Tap on/off + mode persisted (tap_settings); body capped at 1 MiB (truncated flag)

## Bootstrap

### ✅ SSH connect wired to actual SSH + bootstrap
- connect command: SSH establish → bootstrap (detect → upload → start) → store transport

### ✅ remote-agent-host uploaded on first connect
- ✅ Raw binary upload via SSH channel (single `data_bytes` call, no base64/chunk/sleep)
- Skips upload if agent already installed (version check)
- Upload speed: ~1 RTT for 1.7MB binary (was 35+ RTT with base64 chunked echo)

### ✅ Upload latency fix — disable Nagle + fuse round-trips
- ✅ `ssh.rs::connect` sets `config.nodelay = true` — Nagle was the real cause of
  ~8s upload (1.8MB / 8s ≈ 228 KB/s, far below bandwidth). Main speedup.
- ✅ Fused upload round-trips: `mkdir -p && cat > tmp` in one channel (new
  `ssh::upload_raw_cmd`), then a single exec for `stat verify + atomic mv + chmod`.
  Was 4 separate SSH channels (4 RTT) → now 2.
- See design.md §4.17

### ⬜ Optional: gzip-compressed upload for high-latency/WAN links
- Binary gzips to ~780KB (−57%). Would help over public internet, not LAN.
- `upload_raw_cmd` already accepts a custom command — pipe `gzip -dc > tmp` on the
  remote and pass locally-gzipped bytes. Deferred: not a bottleneck post-Nagle fix.

### ⬜ Optional: replace 50ms recv() polling with awaited recv
- spawn_session / relay_session_output poll `try_recv()` + sleep 50ms, adding up to
  50ms latency per message round-trip ("read session" feel).
- Blocked on: spawn-timeout counter assumes 50ms ticks (600 ticks = 30s); changing
  recv() to block would need that timeout reworked. Low payoff, deferred.

### ⬜ Bootstrap progress events in frontend
- `bootstrap:progress` events are emitted but not displayed in AgentManager connect form
- User sees hardcoded log messages instead of real progress

### ⬜ Agent version upgrade on reconnect
- Currently only uploads if not installed; no version comparison
